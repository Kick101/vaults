## 14 enterprise-specific tactics
| **Tactic**              | **Tactic ID** | **Description**                                                                     |
| ----------------------- | ------------- | ----------------------------------------------------------------------------------- |
| 1. Reconnaissance       | TA0043        | Gathering information to plan future attacks, including target profiling.           |
| 2. Resource Development | TA0042        | Establishing resources (e.g., infrastructure, tools, identities) for operations.    |
| 3. Initial Access       | TA0001        | Gaining entry into the target environment (e.g., phishing, exploiting public apps). |
| 4. Execution            | TA0002        | Running malicious code within the victim environment.                               |
| 5. Persistence          | TA0003        | Maintaining a foothold on systems across reboots or credential changes.             |
| 6. Privilege Escalation | TA0004        | Gaining higher-level permissions (e.g., admin/root access).                         |
| 7. Defense Evasion      | TA0005        | Avoiding detection or mitigation (e.g., disabling AV, obfuscation).                 |
| 8. Credential Access    | TA0006        | Stealing account credentials (e.g., via keyloggers, dumps, brute force).            |
| 9. Discovery            | TA0007        | Identifying system, network, and domain details to inform lateral movement.         |
| 10. Lateral Movement    | TA0008        | Moving through the environment to other systems or accounts.                        |
| 11. Collection          | TA0009        | Gathering data of interest from compromised systems.                                |
| 12. Command and Control | TA0011        | Communicating with compromised systems remotely (e.g., C2 servers).                 |
| 13. Exfiltration        | TA0010        | Transferring stolen data out of the target environment.                             |
| 14. Impact              | TA0040        | Disrupting, destroying, or manipulating systems or data (e.g., ransomware).         |

## Use Cases In Sec Ops

| **Use Case**                           | **Explanation**                                                                                 |
| -------------------------------------- | ----------------------------------------------------------------------------------------------- |
| **Detection & Response**               | Enables creation of detection rules and incident response strategies based on known TTPs.       |
| **Security Evaluation & Gap Analysis** | Identifies current security strengths and weaknesses; helps prioritize control investments.     |
| **SOC Maturity Assessment**            | Measures an organization's capability to detect/respond to various techniques; identifies gaps. |
| **Threat Intelligence**                | Provides a standardized language for sharing intelligence internally and externally.            |
| **Threat Intelligence Enrichment**     | Adds context (TTPs, IOCs, targets) to threat feeds for more informed decision-making.           |
| **Behavioral Analytics**               | Supports creation of detection models based on typical attack behaviors.                        |
| **Red Teaming & Pen Testing**          | Guides simulation of real-world attacker behavior during tests.                                 |
| **Training & Education**               | Offers a well-structured resource to train security professionals on attacker methods.          |
