**Cloud Security Posture Management (CSPM)** - In Defender for Cloud, the posture management features provide:

- _Visibility_ is delivered through features like **Secure Score**, which gives a clear view of your security posture.
- *Hardening guidance* - to help you efficiently and effectively improve your security

**Security posture management includes:**

- Security recommendations
- Secure Score
- Regulatory Compliance
- Azure Security Benchmark

>*Secure Score* in Defender for Cloud continuously assesses your environment and aggregates security findings into a single score that reflects your overall risk posture—the higher the score, the lower the risk.

**Cloud Workload Protection (CWP)** focuses on **threat detection and protection** using Microsoft Threat Intelligence and Defender plans tailored to specific workloads (e.g., VMs, Storage, Kubernetes).

>- **CSPM = Visibility (Secure Score) + Hardening Guidance**
> - **CWP = Threat Detection + Workload Protection (via enhanced security features Defender plans)**

>In cloud security, a *workload* refers to any application, service, or resource running in the cloud that performs computing tasks. It can be as simple as a web app or as complex as a distributed microservices architecture.

### 👩‍🏭 Workloads

| Workload Type      | Examples                                                                     |
| ------------------ | ---------------------------------------------------------------------------- |
| **Compute**        | Azure Virtual Machines, Azure App Service, AKS (Kubernetes), Azure Functions |
| **Data**           | Azure SQL Database, Cosmos DB, Azure Storage                                 |
| **Networking**     | Azure Firewall, Application Gateway                                          |
| **Identity**       | Azure Active Directory, Key Vault                                            |
| **Containers**     | Kubernetes workloads (AKS), containerized apps                               |
| **Hybrid/on-prem** | Windows/Linux servers managed via Azure Arc                                  |

### ☁️ Defender Plans:

| Defender Plan                | Purpose                                                                                                  |
| ---------------------------- | -------------------------------------------------------------------------------------------------------- |
| **Defender for Servers**     | Protects Azure and hybrid/on-premises VMs with EDR, vulnerability assessments, Just-In-Time access, etc. |
| **Defender for Storage**     | Detects malware uploads, access anomalies, and suspicious activity in Azure Storage.                     |
| **Defender for SQL**         | Protects Azure SQL Database and SQL on VMs with threat detection and vulnerability assessment.           |
| **Defender for App Service** | Identifies attacks targeting your web apps hosted on Azure App Service.                                  |
| **Defender for Kubernetes**  | Monitors Kubernetes clusters (AKS and self-managed) for runtime threats and misconfigurations.           |
| **Defender for Key Vault**   | Detects unusual and potentially harmful access patterns to Azure Key Vault.                              |

>To extend protection to virtual machines and SQL databases in other clouds or on-premises, deploy *Azure Arc* and enable Defender for Cloud. Azure Arc for servers is a free service, but services used on Arc enabled servers, such as Defender for Cloud, are charged per the pricing for that service.
