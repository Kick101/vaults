### **How Secure Score Is Calculated**

- Secure Score is **control-based**, not recommendation-based.
- **Security Control** = Group of related recommendations targeting a specific **attack surface**.
- You **only gain score** when **all recommendations within a control are remediated** **per resource**.

>For example:  
If a control focuses on **securing management ports**, and there are three related recommendations, you must fix **all three for a resource** to earn points under that control.

## **Understanding a Recommendation**

Each recommendation includes:

- **Severity Indicator** (High, Medium, Low)
    
- **Freshness Interval** (Last assessment timestamp)
    
- **MITRE ATT&CK Mapping** (Where applicable)
    
- **Policy Definition Link** (to view or edit the Azure Policy)
    
- **Open Query** (Azure Resource Graph Explorer query)
    
- **Remediation Steps** (manual or automated)
    

### **Affected Resources Tabs**

- **Healthy**: Issue remediated or not relevant
    
- **Unhealthy**: Resources needing remediation
    
- **Not Applicable**: No assessment possible (with reasons)
    

### **Related Recommendations**

- **Prerequisite**: Must be completed first
    
- **Dependent**: Depends on current one
    
- **Alternative**: Different way to achieve the same goal
    
### Key Built-in Workbooks in Defender for Cloud

 - **Secure Score Over Time**: Track subscription security scores and recommendation changes.
    
- **System Updates**: View missing updates categorized by resource, OS, and severity.
    
- **Vulnerability Assessment Findings**: Analyze results from vulnerability scans.
    
- **Compliance Over Time**: Monitor compliance status against selected regulatory or industry standards.
    
- **Active Alerts**: View current alerts by severity, type, tag, MITRE ATT&CK tactic, and location.
