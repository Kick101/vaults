## SoC Roles

| **Role**                               | **Responsibilities**                                                                                      |
| -------------------------------------- | --------------------------------------------------------------------------------------------------------- |
| **SOC Director**                       | Oversees strategy, budgeting, staffing, and alignment of SOC with organizational goals                    |
| **SOC Manager**                        | Manages daily SOC operations, incident coordination, team leadership, and interdepartmental collaboration |
| **Tier 1 Analyst**                     | Monitors alerts, triages events, and escalates suspicious activity                                        |
| **Tier 2 Analyst**                     | Conducts deeper investigations, identifies threat patterns, and recommends mitigation                     |
| **Tier 3 Analyst**                     | Handles advanced threats, performs threat hunting, and enhances detection capabilities                    |
| **Detection Engineer**                 | Creates and tunes detection rules, improves visibility, and closes detection gaps                         |
| **Incident Responder**                 | Leads incident response, conducts forensic analysis, and restores affected systems                        |
| **Threat Intelligence Analyst**        | Collects and analyzes threat intel to inform proactive defense strategies                                 |
| **Security Engineer**                  | Builds and maintains security infrastructure, provides technical SOC support                              |
| **Compliance & Governance Specialist** | Ensures regulatory compliance, manages audits, and enforces security policies                             |
| **Security Awareness Coordinator**     | Develops employee training and awareness programs to promote secure behavior                              |

## SoC Evolution

__SoC Evolution__
- Availability Monitoring
- Reactive Monitoring
- Proactive Monitoring
- Proactive Automation

|**SOC Generation**|**Key Features**|**Challenges**|**Focus Areas**|
|---|---|---|---|
|**SOC 1.0**|- Basic security tools- Focus on network/perimeter security- Limited integration|- Uncorrelated alerts- Tool sprawl- Manual processes- Lack of visibility beyond network|- Network security- Perimeter defense|
|**SOC 2.0**|- Integrated threat intelligence- Layer-7 and anomaly detection- Botnet & malware analysis|- Still maturing integration- Gaps in collaboration- Limited pre/post-incident learnings|- Situational awareness- Threat intel integration- Pre/post-event management|
|**Cognitive SOC**|- AI/ML-based decision support- Business-context aware- Learning systems- Incident response standardization|- Operational inexperience- Limited cross-team collaboration- Evolving detection logic|- Adaptive defense- Automation- Business-driven threat detection- Continuous learning|
