### Summary

| Constraint         | Description                                                                                           |
| ------------------ | ----------------------------------------------------------------------------------------------------- |
| **NOT NULL**       | Ensures a column **cannot** have `NULL` values.                                                       |
| **UNIQUE**         | Ensures **all values** in a column are **different**.                                                 |
| **PRIMARY KEY**    | Uniquely identifies each record in a table. (Combination of `NOT NULL` and `UNIQUE`)                  |
| **FOREIGN KEY**    | Ensures referential integrity by linking to a **primary key** in another table.                       |
| **CHECK**          | Ensures that values in a column **meet a specific condition**.                                        |
| **DEFAULT**        | Sets a **default value** for a column if no value is provided.                                        |
| **AUTO_INCREMENT** | Automatically generates a **unique number** (often used for primary keys). _(MySQL specific)_         |

```SQL
CREATE TABLE employees (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(100) UNIQUE,
  salary DECIMAL(10,2) CHECK (salary > 0),
  department_id INT,
  hire_date DATE DEFAULT CURDATE(),
  FOREIGN KEY (department_id) REFERENCES departments(id)
);
```

#### When to use Modify & Constraint 

| Task                       | `MODIFY` | `ADD CONSTRAINT` |
| -------------------------- | -------- | ---------------- |
| Make column NOT NULL       | ✅        | ❌                |
| Set a DEFAULT value        | ✅        | ❌                |
| Add UNIQUE constraint      | ❌        | ✅                |
| Add FOREIGN KEY constraint | ❌        | ✅                |
#### ❌ `MODIFY` is for:
- Changing data types
- Adding/removing `NOT NULL`
- Setting `DEFAULT` values
---
### PRIMARY KEY
- Ensures that the column(s) uniquely identify a record in the table.
- **No duplicates** and **no null values** allowed.

__Add primary key__
```SQL
ALTER TABLE products ADD PRIMARY KEY (id);
```


>#### Composite key is a primary key made of *multiple columns*

```SQL
CREATE TABLE course_enrollments (
  student_id INT,
  course_id INT,
  enrollment_date DATE,
  PRIMARY KEY (student_id, course_id)
);
```

---
### FOREIGN KEY
- Ensures data integrity by ensuring that the value in the column matches a value in another table’s primary key column.
- Because `FOREIGN KEY` and `REFERENCES` both allow **multiple columns** (i.e., composite keys), ___parentheses___ are always used for clarity and correctness.

> #### ❌ `departments.id` — *Not valid in REFERENCES*
>
This is used in **queries** like `SELECT` or `JOIN`, but **not** in `FOREIGN KEY` definitions.

__Add foreign key__
```SQL
ALTER TABLE employees 
ADD FOREIGN KEY (department_id) 
REFERENCES departments(id);
```

```SQL
ALTER TABLE employees
ADD CONSTRAINT fk_dept
FOREIGN KEY (department_id) REFERENCES departments(id);
```

__Drop__
>- When dropping a foreign key constraint, you need to refer to the *constraint name*, not the column directly. 
>- The constraint name is usually either set manually when the foreign key is created or automatically generated by the database.

```SQL
ALTER TABLE employees DROP FOREIGN KEY fk_dept;
```

### UNIQUE
```SQL

```

### NOT NULL
```SQL
ALTER TABLE employees
MODIFY name VARCHAR(100) NOT NULL;
```

### CHECK
```SQL
ALTER TABLE employees
ADD CONSTRAINT check_salary CHECK (salary > 0);
```
---
### DEFAULT
```SQL
ALTER TABLE employees
MODIFY department VARCHAR(50) DEFAULT 'General';
```

#### AUTO_INCREMENT
- Reset `AUTO_INCREMENT`: Sometimes, you may want to reset the sequence if the numbers become too large, using:
```SQL
ALTER TABLE products AUTO_INCREMENT = 101; -- Starting From '101'
```
- Specify the starting value:
```SQL
CREATE TABLE products (
  product_id INT AUTO_INCREMENT PRIMARY KEY,
  product_name VARCHAR(100),
  price DECIMAL(10, 2)
) AUTO_INCREMENT = 1001; -- Here!
```
